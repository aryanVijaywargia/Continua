## Design Decisions Made

### CodeRabbit Test Coverage
- **Decision**: Remove test file exclusions (`!**/*_test.go`, `!**/*.test.ts`, `!**/*.spec.ts`)
- **Why**: Tests can have bugs, quality issues, and security problems. Excluding them from review was a temporary choice that shouldn't persist.

### CI Failure Policy
- **Decision**: Add explicit policy to CLAUDE.md prohibiting test bypassing
- **Why**: User explicitly requested this to prevent future incidents where tests are skipped rather than fixed. RCA-first approach ensures actual issues are addressed.

### E2E CI Approach
- **Decision**: Use GitHub Actions services (native PostgreSQL container) over docker-compose
- **Why**: User chose this approach for cleaner integration with GitHub Actions. Native services are well-supported and simpler than managing docker-compose in CI.

### E2E Workflow Structure
- **Decision**: Run the existing Python SDK `e2e_demo.py` as the E2E test
- **Why**: Reuses existing code that already creates traces, makes API calls, and verifies data. No need to create separate E2E test suite.

### E2E Workflow Details
The new workflow:
1. Spins up PostgreSQL 16 Alpine via GitHub Actions services
2. Builds the Go server
3. Runs database migrations
4. Starts server and waits for health check (30 second timeout)
5. Installs Python SDK dependencies via uv
6. Runs `e2e_demo.py`
7. Cleans up server process
8. Collects logs on failure

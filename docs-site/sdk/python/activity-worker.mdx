---
title: "ActivityWorker"
description: "Run engine activity handlers — register Python functions, poll the engine for tasks, dispatch with heartbeats."
---

<Warning>
  `ActivityWorker` is part of Continua's **durable engine surface**, which is currently
  scaffolded. The Python worker is implemented and usable, but the engine itself is not
  the primary product path today. See [Architecture overview](/concepts/overview).
</Warning>

`ActivityWorker` polls a Continua engine endpoint for activity tasks, dispatches them to
registered Python handlers, and reports outcomes back. It handles concurrency,
heartbeats, signal handling, and cancellation lifecycle.

## Quick start

```python
from continua import ActivityWorker

worker = ActivityWorker(
    api_key="...",
    endpoint="http://localhost:8080",
)


@worker.activity("send_email")
def send_email(args: dict) -> dict:
    # do work
    return {"message_id": "abc"}


worker.run()  # blocks until stopped
```

## Constructor

```python
ActivityWorker(
    *,
    api_key: str,
    endpoint: str = "http://localhost:8080",
    worker_id: str | None = None,
    poll_interval: float = 1.0,
    max_concurrent: int = 8,
    heartbeat_interval: float = 10.0,
    install_signal_handlers: bool = True,
)
```

| Parameter | Description |
| --- | --- |
| `api_key` | Engine API key. |
| `endpoint` | Engine server URL. |
| `worker_id` | Stable identifier for this worker process. Auto-generated if omitted. |
| `poll_interval` | Seconds between empty polls. |
| `max_concurrent` | Maximum in-flight tasks. |
| `heartbeat_interval` | Default heartbeat cadence per task. The engine may override per-task. |
| `install_signal_handlers` | Install SIGINT/SIGTERM handlers for graceful shutdown. |

## Registering handlers

### Decorator

```python
@worker.activity("send_email")
def send_email(args: dict) -> dict:
    return {"ok": True}
```

### Imperative

```python
def send_email(args: dict) -> dict:
    return {"ok": True}

worker.register("send_email", send_email)
```

Handlers may optionally accept an `ActivityTaskContext` as a second argument:

```python
from continua import ActivityTaskContext

@worker.activity("long_running")
def long_running(args: dict, ctx: ActivityTaskContext) -> dict:
    for chunk in args["chunks"]:
        if ctx.cancelled():
            raise RuntimeError("cancelled")
        process(chunk)
    return {"ok": True}
```

The worker inspects the handler signature with `inspect.signature` and only passes the
context if the handler declares a second parameter.

## `ActivityTaskContext`

| Attribute / method | Description |
| --- | --- |
| `task_id: str` | The task identifier from the engine. |
| `activity_key: str` | The unique activity key for this task instance. |
| `activity_type: str` | The registered handler type (e.g. `"send_email"`). |
| `lost_lease() -> bool` | `True` if the worker lost the lease (heartbeat failure). |
| `cancelled() -> bool` | `True` if the engine signaled cancellation. |
| `is_cancelled() -> bool` | Alias for `cancelled()`. |

Handlers should check `ctx.cancelled()` periodically during long work and exit promptly.

## Running

### `run()`

Blocks the calling thread, polling and dispatching tasks until `stop()` is called or a
signal is received.

```python
worker.run()
```

### `poll_once() -> int`

Single poll cycle. Returns the number of tasks dispatched. Useful for tests.

```python
dispatched = worker.poll_once()
```

### `stop()`

Signal the worker to exit. Safe to call from another thread or a signal handler.

### `close()`

Release resources (HTTP client, thread pool). Called automatically on `run()` exit.

### `wait_for_idle(*, timeout=None) -> bool`

Block until no tasks are in flight. Returns `True` if the worker reached idle within the
timeout, `False` otherwise. Useful in tests after `poll_once()`.

## Reporting outcomes

Handler return values become the activity output. Raising signals failure:

- `raise NonRetryableError("reason")` — engine marks the task **failed without retry**
- Any other exception — engine marks the task failed and may retry per its policy

```python
from continua import NonRetryableError

@worker.activity("validate_email")
def validate_email(args: dict) -> dict:
    if "@" not in args["email"]:
        raise NonRetryableError("Invalid email format")
    return {"valid": True}
```

## Concurrency

The worker dispatches up to `max_concurrent` tasks concurrently via a thread pool. Each
task gets its own heartbeat timer. The worker drains in-flight tasks before exiting.

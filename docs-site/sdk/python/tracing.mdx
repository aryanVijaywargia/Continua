The SDK exposes three context managers (also usable as decorators): `trace`, `span`, and
`session`. They handle context propagation, parent/child links, and lifecycle events for
you.

## `@trace`

Wraps a function as a top-level trace. The wrapped function becomes a unit of work the
debugger shows in the traces list.

```python
from continua import trace

@trace()
def my_agent(query: str) -> str:
    return "response"


@trace(name="custom_name", tags=["production"])
def another_agent():
    pass
```

### Signature

```python
def trace(
    name: str | None = None,
    *,
    session_id: str | None = None,
    user_id: str | None = None,
    tags: list[str] | None = None,
    metadata: dict[str, Any] | None = None,
) -> Callable[[Callable[..., T]], Callable[..., T]]: ...
```

| Parameter | Type | Description |
| --- | --- | --- |
| `name` | `str \| None` | Trace name. Defaults to the function name. |
| `session_id` | `str \| None` | Pin this trace to a specific session. Usually inherited from `@session` instead. |
| `user_id` | `str \| None` | Optional user identifier. |
| `tags` | `list[str] \| None` | Optional tags for filtering in the debugger. |
| `metadata` | `dict[str, Any] \| None` | Free-form metadata. |

### Using as a context manager

```python
from continua import TraceContext

with TraceContext(name="manual_trace") as t:
    t.set_input({"query": "..."})
    # ...
    t.set_output({"answer": "..."})
```

### `TraceContext` methods

| Method | Description |
| --- | --- |
| `set_input(value)` | Attach the trace-level input payload. |
| `set_output(value)` | Attach the trace-level output payload. |
| `set_metadata(key, value)` | Add a metadata key. |

### Accessing the current trace

```python
from continua import get_current_trace

current = get_current_trace()  # TraceContext | None
```

## `span`

Open a span as a context manager. Spans nest automatically: a `span()` opened inside
another `span()` is a child of the outer one.

```python
from continua import span

with span("openai_call", kind="llm") as s:
    s.set_input({"prompt": "..."})
    result = call_openai()
    s.set_output(result)
```

### Signature

```python
def span(
    name: str,
    *,
    kind: SpanKind = "default",
    metadata: dict[str, Any] | None = None,
) -> SpanContext: ...
```

| Parameter | Description |
| --- | --- |
| `name` | Span name (required). |
| `kind` | Span role hint. See the table below for the full set. |
| `metadata` | Optional metadata dictionary. |

#### Span kinds

`SpanKind` is a literal type defined in `continua.span`. Valid values:

| Kind | Use for |
| --- | --- |
| `"llm"` | A direct LLM call (model inference). |
| `"tool"` | A tool / function invocation made by the agent. |
| `"agent"` | A nested agent or sub-agent loop. |
| `"chain"` | A sequence of steps composed as a unit (e.g. a LangChain-style chain). |
| `"retrieval"` | A retrieval / search step (vector store, BM25, etc.). |
| `"embedding"` | An embedding model call. |
| `"generation"` | A non-chat generation step (image, audio, structured output). |
| `"default"` | Anything that doesn't fit the categories above. |

### `SpanContext` event methods

The full list lives on [Span events](/sdk/python/span-events). Headline items:

- `set_input`, `set_output`
- `set_model`, `set_tokens`, `set_cost`
- `set_llm_response`, `set_tool_call` (emit implicit `effect` events)
- `log`, `error`, `exception`, `metric`
- `state_change`, `decision`, `effect`, `wait`, `snapshot_marker`

### Accessing the current span

```python
from continua import get_current_span

current = get_current_span()  # SpanContext | None
```

## `session`

`@session` groups traces under a shared external identifier and lives on its own
reference page:

→ See [Sessions](/sdk/python/sessions) for the signature, propagation rules, manual
linking, and common pitfalls.

```python
from continua import session, trace

with session("user-conversation-42"):
    @trace(name="turn_1")
    def turn_1(): ...

    turn_1()
```

## How context propagates

The SDK uses Python `contextvars` internally:

- `@trace` sets the current trace for the duration of the wrapped call.
- `span()` reads the current trace + parent span and links itself accordingly.
- `session()` sets the current session, which `@trace` reads on entry.

This means context propagates correctly across function calls, including async code that
preserves `contextvars`. Threads need explicit context propagation.

## Common patterns

### Manual trace boundary

```python
from continua import TraceContext, span

def run():
    with TraceContext(name="run", tags=["batch"]) as t:
        t.set_input({"job_id": "abc"})
        with span("step_1") as s:
            s.set_output({"ok": True})
        t.set_output({"status": "done"})
```

### Nested sessions are not supported

See [Sessions → common pitfalls](/sdk/python/sessions#common-pitfalls).

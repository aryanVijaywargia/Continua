### Existing SQLC Query

-- name: UpdateTraceRollups :exec
UPDATE traces SET
    total_spans = $2,
    total_tokens = $3,
    total_cost = $4,
    error_count = $5,
    duration_ms = CASE WHEN end_time IS NOT NULL AND start_time IS NOT NULL
        THEN EXTRACT(EPOCH FROM (end_time - start_time)) * 1000
        ELSE duration_ms END,
    updated_at = NOW()
WHERE id = $1;

## Recommendations

### Spec 1: Server Bootstrap
**Approach**: Wire existing code with minimal Fx modules
- Create config loading from env vars
- Create pgxpool provider
- Create store provider
- Create API server provider
- Wire Chi router with generated handlers
- Add lifecycle hooks for graceful shutdown

### Spec 2: Auth Enforcement
**Approach**: Add middleware using existing store method
- `GetProjectByAPIKey` already exists in store
- Create middleware that extracts API key from headers
- Inject project ID into request context
- Apply to all routes except `/api/health`

### Spec 3: Python SDK
**Approach**: Build on existing types.py
- Add `client.py` with httpx
- Add `trace.py` with decorator and context manager
- Add `span.py` with context manager
- Add `batch.py` for batching
- Use existing Pydantic types for validation

### Spec 4: Trace Rollups
**Approach**: Compute inline after ingest
- Schema already has rollup columns
- `UpdateTraceRollups` query exists
- Add aggregation query (SUM from spans)
- Call in ingest service after span inserts

### Specs 5-6: Web UI
**Approach**: Use existing stack
- TanStack Query already installed
- Router already configured
- Create API client with fetch + API key header
- Build TracesPage and TraceDetailPage components

The Settings page at `/settings` adapts to the runtime auth mode of your deployment.

## Modes

**Local self-host (no Auth0, no public demo)**

- A **General** card explains the mode.
- An **API keys** card lets you paste, save, and clear the project API key the UI uses
  for read calls. Stored in `localStorage`.
- An **Appearance** card lets you pick System / Light / Dark theme.

**Auth0 operator login** (`AUTH0_ENABLED=true`)

- A **General** card shows the operator account (email, name).
- A **Sign out** button calls Auth0's `logout`.
- The Appearance card behaves the same as in local mode.
- The API keys card is hidden: operator auth doesn't use a local API key.

**Public demo mode** (`PUBLIC_DEMO_ENABLED=true`)

- The Settings page redirects to `/dashboard`. Demo mode is read-only and has no user
  preferences to manage.

## Theme

Three modes: **System**, **Light**, **Dark**. System follows the OS-level preference
via `prefers-color-scheme` and re-evaluates when the OS preference changes.

Theme state is stored in `localStorage` and applied as a class on `<html>`. Tailwind's
dark mode classes (`dark:`) drive the actual styling.

## Managing the local API key

When the deployment runs in local self-host mode, the API keys card is where you wire
up read access:

1. Create a project from the [Projects page](/guides/managing-projects). The UI shows
   the plaintext key **once**.
2. Copy the key into the Settings → API keys card and save.
3. The UI uses the saved key as `X-API-Key` on every read call.

Clear the key with the **Clear key** button. This signs the local UI out, but it does
not revoke the key server-side. To revoke, use the project rotate-key flow.

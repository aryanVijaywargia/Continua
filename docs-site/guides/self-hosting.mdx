Continua is designed to be self-hosted. The server is a single Go binary backed by
Postgres. There is no required SaaS dependency.

## Required configuration

The server reads configuration from environment variables (see
`internal/config/config.go`).

| Variable | Purpose |
| --- | --- |
| `DATABASE_URL` | Postgres connection string. **Required.** |

## Common optional variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `HOST` | `0.0.0.0` | Server bind host |
| `PORT` | `8080` | Server bind port |
| `INGEST_TRUE_ASYNC_DEFAULT` | `false` | Server default async ingest behavior |
| `INGEST_DEPENDENCY_RETRY_WINDOW` | `15m` | Retry window for async ingest dependency resolution |
| `INGEST_FAILED_PAYLOAD_RETENTION` | `7d` | How long to retain failed async ingest payloads |
| `RIVER_QUEUE_INGEST_WORKERS` | `4` | Ingest batch queue concurrency |
| `RIVER_QUEUE_ROLLUP_WORKERS` | `10` | Rollup job concurrency |
| `RIVER_QUEUE_MAINTENANCE_WORKERS` | `1` | Cleanup / maintenance job concurrency |
| `RIVER_QUEUE_DEFAULT_WORKERS` | `1` | Catch-all queue concurrency |

### Demo mode

| Variable | Default | Purpose |
| --- | --- | --- |
| `PUBLIC_DEMO_ENABLED` | `false` | Enables read-only seeded demo mode |
| `PUBLIC_DEMO_PROJECT_ID` | unset | UUID of the project shown in public demo mode (required when demo enabled) |
| `PUBLIC_DEMO_LABEL` | unset | Demo workspace label in the UI |

### Auth0 operator login

Enable Auth0 to gate the debugger UI behind an allow-listed set of operator emails. See
[Auth0 setup](/guides/auth0-setup) for the full walk-through.

| Variable | Default | Purpose |
| --- | --- | --- |
| `AUTH0_ENABLED` | `false` | Master switch |
| `AUTH0_DOMAIN` | unset | Auth0 tenant domain (e.g. `your-tenant.us.auth0.com`) |
| `AUTH0_CLIENT_ID` | unset | SPA client ID |
| `AUTH0_AUDIENCE` | unset | API audience identifier |
| `AUTH0_ALLOWED_EMAILS` | unset | Comma-separated allow-list. At least one required when enabled |

### Engine surface (preview)

The durable engine surface ships behind a preview flag. See
[Engine foundation](/concepts/engine-foundation) for what's implemented and
[Roadmap](/roadmap) for what isn't.

| Variable | Default | Purpose |
| --- | --- | --- |
| `ENGINE_PUBLIC_API_ENABLED` | `false` | Expose `/v1/engine/*` endpoints. Requires `X-Continua-Engine-Preview` on requests |
| `ENGINE_DATABASE_URL` | falls back to `DATABASE_URL` | Optional separate Postgres for the engine schema |
| `ENGINE_PROJECTION_RETENTION_AFTER` | unset | How long to retain projection state after a run reaches terminal state |
| `ENGINE_HISTORY_RETENTION_AFTER` | unset | Retention window for run history. Must be greater than `ENGINE_PROJECTION_RETENTION_AFTER` |

<Warning>
  Do not use `config.example.yaml` as the runtime contract. It is forward-looking and not
  the active config source. Env vars are authoritative.
</Warning>

## Migrations

Apply migrations on the same binary you ship:

```bash
DATABASE_URL=... continua migrate up
```

Migrations are immutable once committed. Create new ones with:

```bash
make migrate-create name=<short_description>
```

## Background workers

River workers run **inside** the platform server process. There is no separate worker
binary to deploy today. The implemented queues are:

- **ingest** — processes accepted async batches
- **rollups** — recomputes trace-level aggregates (status, duration, cost, token counts)
- **cleanup** — deletes processed async payload blobs after retention

Scale by increasing process count or tuning `RIVER_QUEUE_*` pool sizes.

## Health check

```bash
curl https://your-continua-host/api/health
```

Returns `{"status":"ok"}` when the server is up and Postgres is reachable.

## Troubleshooting

### Port `8080` is already in use

```bash
lsof -nP -iTCP:8080
make docker-down
```

### Docker data looks stale

```bash
make reset-demo
```

### App is up but no traces appear

```bash
make docker-logs
docker compose -f deploy/docker-compose/docker-compose.yml --profile seed run --rm demo-seed
```

### Native server fails with `DATABASE_URL environment variable is required`

```bash
export DATABASE_URL="postgres://continua:continua@localhost:5432/continua?sslmode=disable"
make migrate
make dev-server
```

### Native UI can't load data

- Confirm the backend is running on `http://localhost:8080`.
- Confirm Vite is running on `http://localhost:3000`.
- Confirm the UI has a project API key in `/settings`, or create the first project
  from `/projects` if the database is empty.

## What is not in scope today

Per the current [architecture baseline](/concepts/overview), these surfaces exist as
scaffolds and **should not** be relied on in production:

- Durable engine workflow execution
- Live WebSocket runtime
- Proxy capture runtime
- Replay execution runtime
- The TypeScript SDK

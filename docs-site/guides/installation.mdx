Continua ships as a single Go binary plus a Postgres database. There are two supported
local setups: the **Docker demo** (one command) and the **native dev console** (live Vite
reload).

## Docker demo

Use this path for first contact, demos, and when you don't want to install Go, Node, or
Python locally.

### Prerequisites

- Git
- Docker with Docker Compose v2
- A free local port `8080`

### Run

```bash
git clone https://github.com/aryanVijaywargia/Continua.git
cd Continua
make demo
```

`make demo` invokes `scripts/demo.sh`, which:

1. Builds the Docker image from `deploy/docker/Dockerfile`.
2. Starts `postgres` and `continua` from `deploy/docker-compose/docker-compose.yml`.
3. Runs `continua migrate up` before the server starts.
4. Starts `continua serve` with public demo mode enabled.
5. Runs the `demo-seed` service, which resets a deterministic demo project and emits
   sample traces through the ingest API.

Open `http://localhost:8080` to see the seeded demo workspace.

Health check:

```bash
curl http://localhost:8080/api/health
```

### Demo defaults

| Setting | Value |
| --- | --- |
| Project ID | `11111111-1111-4111-8111-111111111111` |
| Demo API key (for seeding) | `continua-demo-local` |
| App URL | `http://localhost:8080` |
| Postgres URL (inside Docker) | `postgres://continua:continua@postgres:5432/continua?sslmode=disable` |

The public demo UI is **read-only**. It's safe to explore without entering an API key.

### Useful commands

```bash
make docker-logs      # follow service logs
make docker-down      # stop the demo stack
make reset-demo       # wipe Docker data volume, rebuild, migrate, reseed
```

## Native dev console

Use this path when you want to iterate on the frontend or ingest your own traces against
a live database.

### Prerequisites

- Go 1.24+
- Node.js 20+
- pnpm 9+
- Docker (for Postgres)
- Python 3.10+ and [`uv`](https://github.com/astral-sh/uv) (for the SDK)

### Run

```bash
./scripts/setup.sh
make dev

export DATABASE_URL="postgres://continua:continua@localhost:5432/continua?sslmode=disable"
make migrate
make dev-server
```

In another terminal:

```bash
make dev-web
```

Open `http://localhost:3000/projects`. For a fresh local database, create your first
project in the UI. Continua shows the generated project API key once; use that key
for SDK ingest and future local console access.

### Emit private traces

```bash
cd sdks/python
CONTINUA_API_URL="http://localhost:8080" \
CONTINUA_API_KEY="<project-api-key>" \
uv run python examples/e2e_demo.py
```

## Ports

| Port | Used by | Path |
| --- | --- | --- |
| `8080` | Go server + embedded UI (Docker demo) | `make demo` |
| `5432` | Postgres | Both setups |
| `3000` | Vite dev server | Native `make dev-web` |

## Runtime configuration

The server reads runtime configuration from environment variables in
`internal/config/config.go`.

Required:

| Variable | Purpose |
| --- | --- |
| `DATABASE_URL` | Postgres connection string |

Common optional variables:

| Variable | Default | Purpose |
| --- | --- | --- |
| `HOST` | `0.0.0.0` | Server bind host |
| `PORT` | `8080` | Server bind port |
| `INGEST_TRUE_ASYNC_DEFAULT` | `false` | Server default async ingest behavior |
| `INGEST_DEPENDENCY_RETRY_WINDOW` | `15m` | Retry window for async batches waiting on dependencies |
| `INGEST_FAILED_PAYLOAD_RETENTION` | `7d` | Retention window for failed async ingest payloads |
| `RIVER_QUEUE_INGEST_WORKERS` | `4` | Ingest batch queue concurrency |
| `RIVER_QUEUE_ROLLUP_WORKERS` | `10` | Rollup job concurrency |
| `RIVER_QUEUE_MAINTENANCE_WORKERS` | `1` | Cleanup / maintenance job concurrency |
| `RIVER_QUEUE_DEFAULT_WORKERS` | `1` | Catch-all queue concurrency |

Engine surface (preview: see [Roadmap](/roadmap) for status):

| Variable | Default | Purpose |
| --- | --- | --- |
| `ENGINE_PUBLIC_API_ENABLED` | `false` | Expose `/v1/engine/*` endpoints. Requires `X-Continua-Engine-Preview` on requests |
| `ENGINE_DATABASE_URL` | falls back to `DATABASE_URL` | Optional separate Postgres for the engine schema |
| `ENGINE_LEASE_COMPLETION_GRACE` | `0` | Optional clock-skew grace for remote activity complete, fail, and retry fences only |
| `ENGINE_PROJECTION_RETENTION_AFTER` | unset | How long to retain projection state after a run reaches terminal state |
| `ENGINE_HISTORY_RETENTION_AFTER` | unset | Retention window for run history. Must be greater than `ENGINE_PROJECTION_RETENTION_AFTER` |

Demo / operator login (optional):

| Variable | Default | Purpose |
| --- | --- | --- |
| `PUBLIC_DEMO_ENABLED` | `false` | Enables read-only seeded demo mode |
| `PUBLIC_DEMO_PROJECT_ID` | unset | UUID of the project shown in public demo mode (required when demo enabled) |
| `PUBLIC_DEMO_LABEL` | unset | Demo workspace label in the UI |
| `AUTH0_ENABLED` | `false` | Enable Auth0 operator login |
| `AUTH0_DOMAIN`, `AUTH0_CLIENT_ID`, `AUTH0_AUDIENCE`, `AUTH0_ALLOWED_EMAILS` | unset | Auth0 tenant configuration (required when `AUTH0_ENABLED=true`). See [Auth0 setup](/guides/auth0-setup) |

`config.example.yaml` is a reference template only. The server does not read YAML
configuration at runtime.

## Troubleshooting

Port `8080` is already in use:

```bash
lsof -nP -iTCP:8080
make docker-down
```

Docker data looks stale:

```bash
make reset-demo
```

The app is up but no traces appear:

```bash
make docker-logs
docker compose -f deploy/docker-compose/docker-compose.yml --profile seed run --rm demo-seed
```

Native server fails with `DATABASE_URL environment variable is required`:

```bash
export DATABASE_URL="postgres://continua:continua@localhost:5432/continua?sslmode=disable"
make migrate
make dev-server
```

Native UI cannot load data:

- Confirm the backend is running on `http://localhost:8080`.
- Confirm Vite is running on `http://localhost:3000`.
- Confirm the UI has a project API key in `/settings`, or create the first project
  from `/projects` if the database is empty.

## Current product boundary

The implemented setup path is:

```text
Python SDK / custom client
  -> authenticated REST ingest
  -> Postgres persistence
  -> River background jobs
  -> REST read APIs
  -> embedded React debugger operator console
```

The TypeScript SDK, live WebSocket runtime, proxy capture, replay runtime, and durable
engine execution are not the main local setup path today.

## Design Decisions Made
- **Pointer parameters for large structs**: Changed store, ingest, and API layer functions to accept pointers instead of values to avoid copying large structs (~300+ bytes). Dereference at sqlc call boundary.
- **Index-based range loops**: Changed from `for _, item := range slice` to `for i := range slice { item := &slice[i] }` to avoid copying large struct values on each iteration.
- **Nolint comments with justification**: Added `//nolint:revive` comments for intentional type stuttering (e.g., `ingest.IngestRequest`) with explanatory comments about why the naming is clearer in import context.
- **Named return parameters**: Added named returns to `processPayload` for clarity: `(jsonData []byte, truncated bool, origSize *int64, reason *string)`.
- **Placeholder SQL for embed**: Created `engine/db/migrations/postgres/0000_placeholder.sql` to satisfy the `//go:embed *.sql` directive in the engine module.
